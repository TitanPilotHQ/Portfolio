# Demo Playbooks — 04. Demo Checklist

## Before the demo

- [ ] Reviewed the prospect's contact-form submission (desk size, asset
      classes, AI usage, governance method — `06-sales-os/01-sales-process.md`,
      Stage 1) to select the right variant (`02-demo-variants.md`).
- [ ] Confirmed which stage of Module 06's sales process this demo
      corresponds to (Stage 4, or a repeat/follow-up demo later in the
      process) — this changes the close (`01-demo-script.md`, section 5).
- [ ] Confirmed no pricing clearance has changed since Module 05 was last
      reviewed (pricing content stays internal-only unless Emad has
      explicitly cleared it for this specific conversation).
- [ ] If this is an investor-variant demo, reviewed Module 07's Investor
      Data Room in full, not from memory (per that module's own governance
      note).

## During the demo

- [ ] Followed `01-demo-script.md`'s section order, using the selected
      variant's emphasis (`02-demo-variants.md`).
- [ ] Did not skip the Devil's Advocate or Risk Gate steps in the Decision
      Walkthrough, even under time pressure.
- [ ] Answered any live objection from a cited source
      (`03-objection-handling.md`) — no improvised claims.
- [ ] Stated next steps explicitly at the close, matching Module 06's
      actual sales process (no implied step that doesn't exist).

## After the demo

- [ ] If a new objection came up with no existing answer, flagged it to
      Emad per Module 06's fallback rule
      (`06-sales-os/03-sales-enablement.md`) — don't let the next demo
      improvise the same gap.
- [ ] Logged the outcome per Module 06's process (currently manual, no CRM —
      `06-sales-os/02-tooling-and-pipeline.md`).
- [ ] If the prospect is proceeding, confirmed which Module 06 stage comes
      next and who owns following up (currently: Emad, per Module 06's
      founder-led process).

## What this checklist is not

A substitute for `01-demo-script.md`, `02-demo-variants.md`, or
`03-objection-handling.md` — this file is a pre-flight/post-flight check,
not a content source. Don't add new claims here; if something needs to be
said during a demo, it belongs in one of those three files, cited from its
canonical source.

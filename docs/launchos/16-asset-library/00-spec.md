# Module 16 — Asset Library — Spec

**Status:** Ready to build (final module — lands last per the master plan's
sequencing rule, since it catalogs the outputs of every other module)
**Depends on:** all 15 prior modules (indexed, not restated) and the two
real visual asset files in `public/`
**Purpose:** The canonical catalog of what LaunchOS actually produced (a
file-level index of all 15 built modules) and Titan Pilot's real visual
assets — without inventing a pitch deck, a one-pager, video, or social
templates that don't exist as real files in this repo.

## Sources

Grounded in: this repo's own `docs/launchos/` directory tree (all 15 built
modules, verified by directory listing, not by memory), `public/logo.png`
and `public/banner.png` (the only two real visual asset files in this
repo), and Module 01's visual identity file (`01-brand-bible/
03-visual-identity.md`) for brand tokens, cited not restated.

## Global Constraint 1 application

This module must not invent an asset that doesn't exist. No pitch deck
file, no one-pager PDF, no video, no social-media template, no letterhead,
no business card, no swag design — none of these exist as real files
anywhere in this repo. If a module's content (e.g., Module 08's demo
script) could theoretically become such an asset, that's a future
production step, not a present asset — this module catalogs what exists
today, not what could be made from existing content.

## Global Constraint 4 application

This module is fundamentally an index — by design, nearly everything it
references is owned by another module or is a real file elsewhere in the
repo. Its own canonical content is limited to: the catalog/index structure
itself, and the two real visual asset files' basic facts (filename,
location, format) that no other module currently documents.

## Structure

| File | Contents |
|---|---|
| `01-launchos-catalog.md` | An index of the 3 program-level files (master plan, intake record, progress ledger) plus all 15 built modules and their files — a table of contents, not a restatement of any module's content |
| `02-visual-assets.md` | The two real visual asset files (`public/logo.png`, `public/banner.png`) — basic facts only, citing Module 01 for brand tokens rather than restating them |
| `03-governance.md` | Review criteria, maintenance triggers, cross-references, honest enforcement note, and an explicit "what this library does not contain" section |

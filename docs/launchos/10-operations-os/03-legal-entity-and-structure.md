# Operations OS — 03. Legal Entity and Structure

## Phase A — today

No legal entity has been incorporated yet. This fact, and its sourcing, is
canonically owned by Module 07 (`07-investor-data-room/01-company-snapshot.md`)
— see that file for the exact citations. This file cites the fact rather
than re-deriving it.

What this means operationally, per Global Constraint 3: Titan Pilot
operates founder-led, informally, under Emad directly. No incorporated
entity means no formal business-registration overhead exists yet — no
entity to file for, and none of the recurring compliance obligations
(annual/franchise filings, registered-agent obligations, entity-level tax
filings) that an incorporated company of any type would carry. This is the
operational consequence of the Phase A fact Module 07 owns, not a new
independent fact — it follows directly from "no entity exists" and isn't
sourced beyond that. It deliberately does not name entity-type-specific
governance mechanics (e.g., board resolutions), since Phase B does not yet
state what entity type this will become.

## Phase B — future (conditional, no date)

Per Global Constraint 3 and Module 07's own Phase B framing: incorporation
will occur before commercial operations begin (`INTAKE-REQUEST.md` §3.4,
via Module 07). Once that happens, an entity structure — a jurisdiction, an
entity type (e.g., LLC, C-Corp, or otherwise), and a legal name — will need
to be established.

None of those specifics are sourced anywhere in this program. No jurisdiction,
entity type, or entity name has been stated by Emad. This file does not
invent one. Stating a plausible default (e.g., assuming Delaware C-Corp
because it's the common choice for a venture-backed US startup) would be
exactly the kind of unsourced, investor-facing-sounding claim Global
Constraint 1 exists to prevent — plausible is not the same as sourced, and
a diligence-grade reader would ask for the source the moment this file
stated a jurisdiction or entity type without one.

## What this file's job is — and isn't

This file's job is to establish that the two phases exist and are properly
separated, per Global Constraint 3: today's informal, no-entity state
(Phase A) as present-tense fact, and the future incorporated state (Phase
B) as conditional, undated plan. Its job is not to guess Phase B's
content. When Emad provides Phase B's specifics — jurisdiction, entity
type, legal name — those get added here as sourced fact, dated to when
Emad stated them, and Phase A's content moves from present-tense fact to
historical framing (per the master plan's stated intent: "so these modules
don't need a rewrite the day incorporation happens — only Phase A needs to
flip to 'historical' and Phase B needs to flip to 'current,'"
`00-launchos-master-plan.md:71-73`).

## What this file deliberately does not invent

- A jurisdiction for future incorporation (e.g., Delaware, another US
  state, another country) — not sourced.
- An entity type (LLC, C-Corp, S-Corp, or otherwise) — not sourced.
- A legal entity name — not sourced. (Titan Pilot is the product/brand
  name in current use; whether that becomes the exact incorporated legal
  name, or the legal name differs, is not stated anywhere.)
- A timeline or date for incorporation — Module 07 states only that it
  will occur "before commercial operations begin," with no date attached,
  and this file does not add one.
- An independent restatement of the Phase A "no entity" fact's sourcing
  detail — that lives in Module 07's file; this file cites it.

# Messaging Bible — 04. Governance

## Ownership

Emad, same rationale as Modules 01-02.

## Review criteria

1. Every channel message is either a verbatim citation or an assembly of
   already-approved fragments — no new claim introduced at assembly time.
2. Every term used matches its glossary definition; no term used outside its
   mandatory companion phrase.
3. No duplication of Module 01's elevator pitches or Module 02's scenes —
   this module references them, it doesn't restate them.
4. Consistent with Global Constraint 3 where a channel message touches
   customer type, pricing, or company stage (e.g., the cold-email subject
   line and ICP framing should stay consistent with Module 06 once built).

## Adversarial review focus

- Does any "assembled from" message quietly introduce a claim that isn't in
  either cited source when the two fragments are combined?
- Does the terminology glossary's "mandatory companion phrase" rule actually
  get followed in this module's own channel messages (self-consistency, same
  test applied to Module 01)?
- Is the "Trader" entry's guidance (default to "desk" over "trader") applied
  consistently across all of `03-channel-messages.md`, or does an older draft
  habit slip through?

## Maintenance triggers

- Any change to Module 01's elevator pitches or Module 02's scenes → re-check
  every channel message that assembles from them
- Any new customer-facing term used twice in shipped copy → add to the
  glossary
- Module 06 (Sales OS) landing → re-verify the ICP framing implied in the
  cold-email subject line against Module 06's actual documented ICP

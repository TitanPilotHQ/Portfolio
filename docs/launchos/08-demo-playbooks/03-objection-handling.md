# Demo Playbooks — 03. Objection Handling

What to do when a live objection comes up mid-demo — distinct from
`01-demo-script.md`'s planned sequence, since objections arrive unscheduled.

## The pattern

1. Don't defer a real objection to "the end" — Module 02's "Discipline
   List" scene principle (state limits proactively) extends to reactive
   moments too: an unanswered objection sitting unaddressed for the rest of
   the demo reads as evasion, not politeness.
2. Answer from an existing source. Never improvise a new claim live — this
   is the demo-specific application of Module 06's sales-enablement rule
   (`06-sales-os/03-sales-enablement.md`, "The one hard rule").
3. If genuinely uncovered, use Module 06's exact fallback: "I don't have
   that answer worked out yet — let me get back to you," or the standard
   disclaimer if it touches trading performance
   (`06-sales-os/03-sales-enablement.md`, same section — not restated here).

## Common objections and where the real answer lives

| Objection | Source |
|---|---|
| "How is this different from [competitor]?" | `04-positioning/02-competitive-positioning-map.md` — find the specific competitor's cluster, use its concede-paired differentiation |
| "This seems like it doesn't do much — where's the automation?" | Module 02's Trust Ladder scene — the current stage is deliberately Shadow, not Copilot or Autonomous; explain why, don't apologize for it |
| "What happens if the AI is wrong?" | Module 02's Failure-Mode Beat scene (already in the core script, section 4 — if asked earlier, pull it forward rather than making the prospect wait) |
| "How much does this cost?" | Module 05's pricing decision tree governs what can be said. The current design-partner phase is free (already public — the one thing the tree's "already public" branch permits restating); beyond that, pricing "isn't finalized yet" (`05-pricing-system/01-pricing-philosophy.md`'s framing) — state it that way, not softened or hardened |
| "Do you have other customers using this?" | Module 01's Company stage section + Module 06's Stage 6 — honest: no confirmed design partner in the claims register yet, currently in the design-partner evaluation phase |
| "Why should I trust a solo-founder company with this?" | Module 01's Company stage + Module 07's Investor FAQ "key-person risk" answer applies here too (honest, not softened) |
| "Is this regulated / compliant?" | No regulatory approval, certification, or endorsement is claimed (`COMPANY_NEVER_CLAIMS`, Module 01) — state this directly rather than deflecting |

## What never changes regardless of objection

The disclaimer requirement (Module 01's pre-publish checklist item, applies
live too) and the never-claims list — an objection is exactly the moment of
highest pressure to overclaim to close the room, which is precisely when
the discipline matters most.

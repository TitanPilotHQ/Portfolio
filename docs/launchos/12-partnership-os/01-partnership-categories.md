# Partnership OS — 01. Partnership Categories

## Source

> Current target partnership categories include brokers, market-data
> providers, cloud infrastructure providers, AI model providers, compliance
> technology providers, and enterprise systems integrators.

*(Source: `INTAKE-REQUEST.md` §7.1, Emad's direct answer, 2026-07-12, quoted
verbatim.)*

This is the complete list. Intake §7.1 states these are "target"
categories — it does not state that Titan Pilot has an active integration,
a technical requirement, a pilot, or any other relationship with any
category, and this file does not add one. Each category below is described
at the general, industry-standard level: what that type of organization
does in the trading-technology industry, and why a company building
AI-assisted trading infrastructure would plausibly consider that category
relevant. None of these descriptions are claims about Titan Pilot's current
product, roadmap, or technical plans — see "What this file deliberately does
not invent" below.

## The six categories

### Brokers

A broker, in this industry, is the entity that holds trading accounts and
executes orders in the market. A partnership in this category would
generically concern account or execution connectivity — the layer between a
trading system's decisions and the market itself.

### Market-data providers

A market-data provider supplies the price, quote, and reference data that a
trading system consumes as input. A partnership in this category would
generically concern the data feeds a trading system needs to observe market
state.

### Cloud infrastructure providers

A cloud infrastructure provider supplies hosting, compute, storage, and
networking as a service. A partnership in this category would generically
concern where and how a system's infrastructure runs.

### AI model providers

An AI model provider supplies the underlying machine-learning or language
models that an AI-powered product can build on. A partnership in this
category would generically concern the models a product like Titan Pilot's
could draw on.

### Compliance technology providers

A compliance technology provider supplies governance, audit, monitoring, or
regulatory-reporting tooling used by financial and trading organizations. A
partnership in this category would generically concern governance or audit
tooling that surrounds a trading system rather than the trading logic
itself.

### Enterprise systems integrators

An enterprise systems integrator is an organization that connects and
deploys software into a larger enterprise's existing technology
environment, typically for large or complex customer organizations. A
partnership in this category would generically concern how a product gets
deployed inside an enterprise customer's environment.

## Relationship to Module 04's competitive map

Module 04 (`04-positioning/02-competitive-positioning-map.md`) documents 8
real, named competitors, organized by the market they compete in. Some of
those named competitors sit in markets that are conceptually adjacent to a
category listed above (for example, a competitor whose market is execution
infrastructure sits conceptually near the broker category). This file does
not cross-reference any competitor from Module 04 as a candidate partner,
and does not treat category adjacency as evidence of partnership interest
in either direction. A "competitor" and a "target partnership category" are
different claims sourced from different places in this program — Module 04
states a competitive relationship that exists today on the live site;
intake §7.1 states a target category with no named party attached.
Conflating the two would assert a relationship that isn't sourced. Readers
of this file should not infer that any named competitor in Module 04 is, or
is being considered as, a partner.

## What this file deliberately does not invent

- No real broker, market-data provider, cloud infrastructure provider, AI
  model provider, compliance technology provider, or systems integrator is
  named anywhere in this file, including as an illustrative example (e.g.,
  "a provider like X") — naming even one, even hypothetically, would imply
  a relationship or evaluation that intake §7.1 does not state.
- No claim that Titan Pilot currently integrates with, evaluates, or has a
  technical requirement involving any category — intake §7.1 states these
  are target categories, nothing about current integration status.
- No priority or sequencing among the six categories — intake §7.1 lists
  them without ranking, and this file preserves that order and that
  flatness.
- No claim that any category maps to a specific product feature or
  architectural component of Titan Pilot's system — product capability
  facts are owned by Titan engineering documentation
  (`00-launchos-master-plan.md`, Global Constraint 4), not this module.
